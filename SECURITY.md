# Security Policy

Opalescent is experimental compiler infrastructure. Security reports are welcome, especially when a bug could make generated programs unsafe, execute unexpected host commands, read or write unintended files, or expose private data.

## Supported versions

The project has not made a stable release yet. Security fixes currently target the `main` branch.

## How to report

Until a dedicated security contact is published, use GitHub's private vulnerability reporting for the repository if available.

Please include:

- A short description of the problem.
- The affected platform.
- The exact command you ran.
- A minimal source file or project that reproduces the issue.
- Whether the issue affects the compiler process, generated program, runtime library, package manager, or editor tooling.

## Coordinated disclosure

The maintainers will acknowledge reports as soon as practical, investigate the affected code path, and coordinate a fix before public disclosure when the issue is exploitable.
