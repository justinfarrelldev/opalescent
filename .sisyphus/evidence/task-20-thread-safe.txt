# Task 2a: Thread-Safe invalid_digit_error Buffer

## Change Applied
File: runtime/opal_runtime.c (lines 22-27)

BEFORE:
```c
static const char* invalid_digit_error(char ch) {
    static char msg[64];
    snprintf(msg, sizeof(msg), "invalid digit '%c' in input", ch);
    return msg;
}
```

AFTER:
```c
static const char* invalid_digit_error(char ch) {
    // Thread-local buffer: each thread gets its own copy to avoid data races
    static _Thread_local char msg[64];
    snprintf(msg, sizeof(msg), "invalid digit '%c' in input", ch);
    return msg;
}
```

## Thread-Safety Mechanism
- Added `_Thread_local` keyword (C11 standard)
- Each thread gets its own independent copy of the `msg` buffer
- Eliminates data race where concurrent threads could overwrite each other's error messages
- No mutex locking needed; storage is thread-local by design
