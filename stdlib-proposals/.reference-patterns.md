# Canonical `.op` Reference Snippets

This document captures 6 canonical code patterns from the language specification that serve as the style authority for all 58 stdlib proposals.

---

### Snippet 1: Public Function with Doc Block and `errors` Clause

**Source**: `language-spec/error_handling_samples.op` line 20

```op
##
    Description: Lower-level parser that may produce a ParseError.
##
let parse_number_core = f(text: string): int32 errors ParseError =>
    return 7
```

This snippet demonstrates the canonical pattern for declaring a public function with a documentation block (`##…##`) and an `errors` clause in the function signature to declare possible error types.

### Snippet 2: `guard … into … else … =>` Pattern

**Source**: `language-spec/error_handling_samples.op` line 26

```op
guard parse_number_core(text) into parsed else _e =>
    log_parse_failure(text)
    return 0
return parsed
```

This snippet shows the canonical `guard` statement for error handling: it attempts to call a function that may error, binds the success value to `parsed`, and provides an `else` branch to handle the error case.

### Snippet 3: `propagate` Pattern

**Source**: `language-spec/error_handling_samples.op` line 40

```op
let content = propagate read_line_from_disk(path)
```

This snippet demonstrates the canonical `propagate` keyword, which forwards errors from a called function directly to the caller without local handling.

### Snippet 4: Type Definition with Constructor Usage

**Source**: `language-spec/types_example.types.op` line 8 and `language-spec/types_usage_example.op` line 9

Type definition:
```op
type Person:
    name: string
    age: int32
```

Constructor usage:
```op
let alice: Person = new Person:
                            name: 'Alice'
                            age: 30
```

This snippet shows the canonical pattern for defining a record type and constructing an instance using the `new TypeName:` syntax with labeled field initialization.

### Snippet 5: Import Statement Examples

**Source**: `language-spec/simple_quiz.op` lines 4-5 and `language-spec/types_usage_example.op` line 1

```op
import take_input, string_to_int32 from standard
import random_int32 from math
```

And:
```op
import Person from ./models.types
```

These snippets demonstrate the canonical import forms present in the language spec:
- **Bare specifier with `from` clause**: `import take_input, string_to_int32 from standard`
- **Relative path import**: `import Person from ./models.types`

### Snippet 6: Array-Returning Function Using `T[]` Syntax

**Source**: `language-spec/array_helpers.op` line 20

```op
let map = f<T, U>(xs: T[], fn: f(T): U): U[] =>
    let mutable out: U[] = []
    for x in xs:
        out.push(fn(x))
    return out
```

This snippet demonstrates the canonical pattern for a generic function that returns an array using the `T[]` syntax, including generic type parameters and array construction with `push`.
